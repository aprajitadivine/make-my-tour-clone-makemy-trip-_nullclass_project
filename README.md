# MakeMyTour — MakeMyTrip Clone (Spring Boot + React)

A MakeMyTrip-inspired real-time travel-booking platform.

| Layer | Technology |
|-------|-----------|
| Frontend | React 18, Vite, Tailwind CSS, React Router v6, Axios |
| Backend | Spring Boot 3, Spring Security 6, JWT, Spring Data JPA |
| Database | H2 in-memory (dev) — swap for MySQL/PostgreSQL in production |
| Frontend hosting | Netlify |
| Backend hosting | Render (or Railway / Heroku / any Docker-capable platform) |

**Live demo:** https://make-my-trip-clone-makemytour.netlify.app/

---

## Features

- Flight & hotel search with dynamic pricing engine
- JWT-based authentication (register / login)
- Booking management, cancellation & refund processing
- Real-time flight status tracking
- Interactive seat map & seat selection
- Review & rating system
- AI-based travel recommendation module

## Local Development

> **Quick-start checklist**
> 1. Start the backend **first** (it seeds sample data on boot)
> 2. Then start the frontend — the Vite proxy sends all `/api` calls to the backend
> 3. No `.env.local` is needed for local development
> 4. Default credentials: **admin / admin-demo-pass** · alice / alice-demo-pass · bob / bob-demo-pass
>    *(Override via `SEED_ADMIN_PASSWORD`, `SEED_ALICE_PASSWORD`, `SEED_BOB_PASSWORD` env vars)*

### Prerequisites
- Java 17+
- Maven 3.8+
- Node.js 18+ & npm 9+

### 1 — Start the backend

```bash
cd backend
mvn spring-boot:run
# API available at http://localhost:8080
# H2 console at  http://localhost:8080/h2-console
```

Sample data (5 flights, 5 hotels, 3 travel stories) is seeded automatically on
startup by `DataInitializer`.  The database uses H2 in-memory by default, so
the data is re-seeded fresh on every restart — no stale state, no disk files.

### 2 — Start the frontend

```bash
cd frontend
npm install
npm run dev
# UI available at http://localhost:5173
```

The Vite dev-server proxies all `/api` requests to `http://localhost:8080`, so
no extra configuration is needed during local development.

### Troubleshooting — "No flights / hotels found"

If the flight and hotel cards are missing after the frontend loads, the backend
is not reachable.  The UI will display an **amber warm-up indicator** while
retrying automatically (3 attempts over ~60 s), then a **red error banner** with
a **Retry** button if all retries fail.

Common causes:
| Symptom | Fix |
|---------|-----|
| Frontend started before backend | Start/restart the backend and click Retry |
| `VITE_API_BASE_URL` set to a sleeping Render URL | The frontend retries automatically; wait ~60 s or click Retry |
| `VITE_API_BASE_URL` points to wrong/renamed service | Update the env var in the Netlify dashboard and redeploy |
| CORS error in browser console | Set `CORS_ALLOWED_ORIGINS` on Render to your exact Netlify URL (no trailing slash) |
| Backend health check fails | Visit `https://<render-url>/api/health` — should return `{"status":"UP"}` |

## Production Deployment

### Overview

```
GitHub repo
  ├── backend/   → deployed to Render  (https://makemytour-backend.onrender.com)
  └── frontend/  → deployed to Netlify (https://make-my-trip-clone-makemytour.netlify.app)
```

The two sides are connected by one environment variable:

| Where | Variable | Value |
|-------|----------|-------|
| Netlify → Environment Variables | `VITE_API_BASE_URL` | `https://<your-render-service>.onrender.com/api` |
| Render → Environment Variables | `CORS_ALLOWED_ORIGINS` | `https://<your-netlify-site>.netlify.app` |
| Render → Environment Variables | `JWT_SECRET` | *(auto-generated by Render)* |

---

### Part A — Deploy the Backend to Render

#### Option 1: One-click via Render Blueprint (recommended)

1. Fork or clone this repository to your own GitHub account.
2. Go to **https://render.com** and sign in (free account is enough).
3. Click **New → Blueprint** and connect your GitHub repo.
4. Render detects `render.yaml` at the repo root and creates the
   **makemytour-backend** web service automatically.
5. Once the build finishes (≈ 3–5 min), click the service URL shown in
   the Render dashboard — it will look like:
   ```
   https://makemytour-backend.onrender.com
   ```
6. Test the health endpoint:
   ```
   https://makemytour-backend.onrender.com/api/health
   ```
   You should see `{"status":"UP"}`.
7. Copy the full service URL — you will need it in Part B.

> **Note:** Render's free tier spins down idle services after ~15 minutes.
> The first request after a spin-down takes **30–60 seconds** to respond.
> The frontend retries automatically up to 3 times (10 s → 20 s → 30 s gaps)
> while showing a warm-up indicator — no action is usually needed.
> If you prefer always-on behaviour, upgrade to the **Starter** plan ($7/month).
>
> **Keep-alive tip:** To prevent cold starts entirely on the free tier, set up an
> external ping service (e.g., [UptimeRobot](https://uptimerobot.com),
> [cron-job.org](https://cron-job.org)) to call
> `https://<your-render-service>.onrender.com/api/health` every **14 minutes**.
> The `/api/health` endpoint is lightweight (no database query) so it adds
> negligible server load.

#### Option 2: Manual service creation on Render

1. **New → Web Service** → connect your GitHub repo.
2. Set the following fields:

   | Field | Value |
   |-------|-------|
   | **Name** | `makemytour-backend` |
   | **Environment** | `Docker` |
   | **Dockerfile path** | `./backend/Dockerfile` |
   | **Docker context** | `./backend` |
   | **Health check path** | `/api/health` |

3. Under **Environment Variables**, add:

   | Key | Value |
   |-----|-------|
   | `JWT_SECRET` | *(click "Generate")* |
   | `CORS_ALLOWED_ORIGINS` | `https://<your-netlify-site>.netlify.app` |

   > If you have a custom Netlify domain, use that URL instead.
   > Multiple origins (e.g., Netlify + local) can be comma-separated:
   > `https://yoursite.netlify.app,http://localhost:5173`

4. Click **Create Web Service** and wait for the deploy to complete.

#### Option 3: Deploy to Railway

1. Go to **https://railway.app** and create a new project from your GitHub repo.
2. Add a new **Service → Docker** pointing to `backend/Dockerfile`.
3. Set the same environment variables as above (`JWT_SECRET`, `CORS_ALLOWED_ORIGINS`).
4. Railway injects `PORT` automatically — the backend is pre-configured to read it.
5. Copy the generated Railway URL (e.g. `https://makemytour-backend.up.railway.app`).

---

### Part B — Connect the Frontend on Netlify

1. Go to your **Netlify site dashboard**
   (https://app.netlify.com → select your site).
2. Navigate to **Site configuration → Environment variables**.
3. Click **Add a variable** and set:

   | Key | Value |
   |-----|-------|
   | `VITE_API_BASE_URL` | `https://makemytour-backend.onrender.com/api` |

   > Replace the hostname with the URL you copied from Render/Railway.
   > The `/api` path suffix is required — all backend endpoints are under `/api`.

4. Go to **Deploys → Trigger deploy → Deploy site** to rebuild the frontend
   with the new variable baked in.
5. After the deploy finishes, open
   https://make-my-trip-clone-makemytour.netlify.app — the site should
   now load flight & hotel data from the live backend.

> **Important:** Do **not** add `VITE_API_BASE_URL` to `netlify.toml`'s
> `[build.environment]` section — it is intentionally absent from source control
> so the backend URL can be updated without a code change.

---

### Part C — Configure CORS on the Backend

The `CORS_ALLOWED_ORIGINS` environment variable on Render must list every
origin that the browser uses to call the backend.

**Single Netlify domain** (most common):
```
CORS_ALLOWED_ORIGINS=https://make-my-trip-clone-makemytour.netlify.app
```

**Multiple origins** (e.g., Netlify + localhost):
```
CORS_ALLOWED_ORIGINS=https://make-my-trip-clone-makemytour.netlify.app,http://localhost:5173
```

After changing this variable in the Render dashboard, click
**Manual Deploy → Deploy latest commit** to restart the service.

## Environment Variable Reference

### Backend (set on Render / Railway / Heroku)

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `8080` | HTTP port — injected automatically by all major PaaS platforms |
| `JWT_SECRET` | *(dev fallback — change in prod)* | Secret key for signing JWT tokens. Use at least 32 random characters. |
| `CORS_ALLOWED_ORIGINS` | `http://localhost:5173,http://localhost:3000` | Comma-separated list of frontend origins allowed by CORS |
| `SEED_ADMIN_PASSWORD` | *(demo default)* | Password for the seeded `admin` user |
| `SEED_ALICE_PASSWORD` | *(demo default)* | Password for the seeded `alice` user |
| `SEED_BOB_PASSWORD` | *(demo default)* | Password for the seeded `bob` user |

### Frontend (set on Netlify)

| Variable | Default | Description |
|----------|---------|-------------|
| `VITE_API_BASE_URL` | *(empty — uses Vite proxy in dev)* | Full base URL of the backend API, e.g. `https://makemytour-backend.onrender.com/api` |

> Copy `frontend/.env.example` to `frontend/.env.local` for local overrides
> (this file is git-ignored).

```
│           └── application.properties

└── render.yaml                     # Render Blueprint for one-click backend deploy
```

## API Endpoints (Quick Reference)

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/health` | — | Service health check (no DB query) |
| POST | `/api/auth/register` | — | Create account |
| POST | `/api/auth/login` | — | Login → JWT |
| GET | `/api/flights` | — | List all flights |
| GET | `/api/flights/search` | — | Search by origin/destination/date |
| GET | `/api/flights/{id}/status` | — | Live flight status |
| GET | `/api/flights/{id}/pricing` | — | Dynamic price for a date |
| GET | `/api/hotels` | — | List all hotels |
| GET | `/api/hotels/search` | — | Search by city/stars |
| POST | `/api/bookings` | ✓ JWT | Create booking |
| GET | `/api/bookings/my` | ✓ JWT | My bookings |
| DELETE | `/api/bookings/{id}/cancel` | ✓ JWT | Cancel & refund |
| POST | `/api/reviews` | ✓ JWT | Submit review |
| GET | `/api/bookings/recommend` | ✓ JWT | AI recommendation |
